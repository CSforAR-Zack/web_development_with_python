Separation of Concerns
-----------------------
- models/        -> data structure & persistence only (SQLAlchemy models, no business logic)
- auth/, todos/  -> each feature is a self-contained blueprint: routes (HTTP layer) call
                    services (business logic layer), which call models (data layer)
- forms.py       -> input validation, isolated from route handlers
- templates/     -> presentation only, organized to mirror the blueprints
- config.py + extensions.py -> app-wide setup kept out of feature code, wired together
                    once in app/__init__.py (application factory pattern)

Request flow example (edit a task):
  routes.py (todos) -> forms.py validates input -> services.py (update_task) ->
  models/task.py (Task) -> db.session.commit() -> redirect back to list.jinja
